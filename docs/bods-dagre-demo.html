Why edges were invisible: bods-dagre's Dr/Pr decorators (1) set opacity:0 on the original dagre edge path and (2) insert a styled clone whose path is recomputed via BezierJS.SVGtoBeziers(d).offset(n). For GLEIF fan-out layouts in TB mode, the offset returns an empty PolyBezier({curves:[]}). The internal Br() function then tries to access curves[0][0].x on an empty array, throwing TypeError, which is swallowed silently inside d3's .each(). The clone path attribute is left at a short stub value. Both paths are invisible — but the edge label ("Controls") is a separate SVG element and still renders, explaining exactly what was visible in the screenshot.

Interest type: otherInfluenceOrControl, beneficialOwnershipOrControl: false. No share percentage → BezierJS offset falls back to 5px, control stroke colour #349aee.

Additional issue: bods-dagre calls document.querySelector("#zoom_in").addEventListener(...) unconditionally after every draw(). If those elements are absent the whole call throws before any edges are rendered — which produced the error you saw opening the previous version of this demo.

Two-pass fix in BODSGraph.tsx (applied here in the graph above):

Pass 1 — unconditionally restore all original dagre edges hidden by Dr/Pr:

el.querySelectorAll('g.edgePath').forEach(edge => {
  const isClone = edge.classList.contains('own') || edge.classList.contains('control');
  if (!isClone && edge.getAttribute('style')?.includes('opacity: 0')) {
    edge.style.opacity = '1';
    const p = edge.querySelector('path');
    if (p && !p.getAttribute('stroke') && !p.style.stroke) {
      p.style.stroke = '#666';
      p.style.strokeWidth = '1.5px';
      p.style.fill = 'none';
    }
  }
});

Pass 2 — remove broken styled clones (NaN, empty, or stub paths):

el.querySelectorAll('g.edgePath.own, g.edgePath.control').forEach(clone => {
  const path = clone.querySelector('path');
  const d = path?.getAttribute('d') ?? '';
  if (!d || d.trim().length < 10 || d.includes('NaN')) clone.remove();
});

Valid clones (properly formed BezierJS offsets) remain and render coloured on top of the restored originals.
