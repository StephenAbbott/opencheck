Diagnosis: why edges are invisible in OpenCheck

Bug analysis

Root cause: bods-dagre's Dr/Pr decorators (1) hide original dagre edges with opacity:0 and (2) insert styled clones whose paths are recomputed via BezierJS SVGtoBeziers().offset(n). When the bezier degenerates — either collinear control points (straight-line edges in simple TB graphs) or an empty PolyBezier returned for wide fan-out layouts — the internal Br() call fails silently inside d3's .each(), leaving the clone path at a stub value. Result: original hidden + clone invisible = no visible edge line, though the edge label still renders.

Fix applied below (right panel): Pass 1 — restore all hidden original dagre edges unconditionally as a guaranteed fallback (grey lines). Pass 2 — remove broken styled clones (NaN or short paths). Valid clones continue to render on top of the restored originals.
